# Rentacar Pro — レンタカー業務管理システム（デモ版）

COMPASS_CAR 様向けに作成した、レンタカー業務管理システムの **フロントエンド・デモ** です。
お問い合わせ〜予約〜契約書類〜お客様対応までを一つの情報でつなぐ、多言語対応の業務システムの完成イメージを、実際の画面操作で確認できます。

> ⚠️ 本デモはフロントエンドのみの試作版です。バックエンド（データ保存・送信・決済・WhatsApp連携）は含まれません。すべてサンプルデータで動作します。

---

## 起動方法

```bash
npm install      # 初回のみ
npm run dev      # 開発サーバー起動 → http://localhost:3000
```

本番ビルドを確認する場合:

```bash
npm run build && npm run start
```

## 画面構成（デモのみどころ）

トップページ（`/`）から「お客様向けサイト」「管理システム」のどちらにも入れます。

### お客様向けサイト
| 画面 | パス | 内容 |
|------|------|------|
| 車両検索 | `/search` | 条件で車両を検索・一覧（多言語・通貨表示） |
| オプション選択 | `/booking/options` | 保険・装備オプション（**選択すると料金が即時に再計算**） |
| お客様情報入力 | `/booking/info` | 予約者・運転者・受取情報の入力フォーム |
| 予約確認 | `/booking/confirm` | 入力内容の最終確認 |
| 予約完了 | `/booking/complete` | 予約番号の発行・今後の流れ |

### 管理システム
| 画面 | パス | 内容 |
|------|------|------|
| ダッシュボード | `/admin` | 予約状況・空車・見積依頼・整備アラートを一覧 |
| 予約管理 | `/admin/reservations` | 車両スケジュール（**予約バーをクリックで詳細表示**） |
| 書類管理 | `/admin/documents` | **一度の入力で4種類の書類を自動生成**（注文確認書／賃貸契約書／車両確認書／個人情報同意書） |
| 見積・車両・整備・顧客・売上・レポート ほか | `/admin/*` | 次フェーズで実装予定の機能の枠を表示 |

### 特にご注目いただきたい点
- **書類自動生成（`/admin/documents`）** — 現在の運用では同じ情報を4つの書類へ転記されていますが、本システムでは予約時の入力から全書類を自動生成します。「書類を自動生成」ボタンで動作をご確認いただけます。
- **多言語対応** — 画面右上の言語切替（日本語／English／中文／한국어）で、UI全体が切り替わります。お客様が触れる情報全体を多言語で扱う構成です。
- **予約スケジュール** — 15台規模の車両を週表示のカレンダーで管理。予約バーをクリックすると右側に詳細が表示されます。

## 技術スタック

| 領域 | 採用技術 |
|------|----------|
| フロント／管理画面 | **Next.js 16（App Router）+ TypeScript** |
| UI | **Tailwind CSS v4** + lucide-react（アイコン） |
| 多言語 | 軽量 i18n（React Context）— 本番では言語ごとの帳票・通知も対応 |
| データ（本番想定） | Supabase（Postgres / Auth / Storage） |
| フォーム（本番想定） | React Hook Form + Zod |
| 書類生成（本番想定） | Word/PDFテンプレートからの自動差し込み |
| WhatsApp（本番想定） | リンク連携 → WhatsApp Business Cloud API + Webhook |
| ホスティング（本番想定） | Vercel（フロント）+ Supabase（バックエンド） |

## 実装ステータス

このデモで形にしているのは、提案の **第1段階（Phase 1）** に当たる画面群です。

- **Phase 1（このデモの範囲）**: ホームページ・予約フロー・管理ダッシュボード・予約スケジュール・顧客／予約管理・書類自動生成・多言語表示
- **Phase 2（次段階）**: WhatsApp Business API 連携、メッセージ履歴、予約情報との紐付け、各種通知の自動化

デモ内のサンプルデータは `lib/mock-data.ts` に集約しています。本番ではこの部分が Supabase 経由の実データに置き換わります。
